Concepts - 1

  • MVVM (Model, View, ViewModel)
    • Model:
      • domain model or data access
      • the same as in MVC
    • View:
      • UI that displays the state of the ViewModel
      • simple HTML document
      • not a "passive" view as in most other frameworks
      • rather an "active" view that displays the state of the ViewModel
      • views contain data-bindings and handle events to the ViewModel
    • ViewModel:
      • exposes data from the model to the view layer
      • handles the view's display and interaction logic

Concepts - 2

  • Declarative data bindings
    • using data-* attributes in the view
    • two-way binding (model is updated when view is updated and vice versa)
    • custom data bindings can be added easily
  • Automatic dependency tracking
    • observable model values
    • computed model values can use observable values and will change if they change
  • nested views / templates
    • models or model values can be displayed with a template
    • if model value changes, only the matching template is re-rendered
    • if model value changes, only the matching DOM element/attribute/content is re-rendered

Simple two-way data binding

<div class="liveExample">
    <p>First name: <input data-bind='value: firstName, valueUpdate: "afterkeydown"' /></p>
    <p>Last name: <input data-bind='value: lastName, valueUpdate: "afterkeydown"' /></p>
    <h2>Hello, <span data-bind='text: fullName'> </span>!</h2>
    <div data-bind="visible: fullName().length > 15">You have a really long name!</div>
</div>
<script type="text/javascript">
    var ViewModel = function(first, last) {
    this.firstName = ko.observable(first);
    this.lastName = ko.observable(last);
    this.fullName = ko.computed(function() {
        // Knockout tracks dependencies automatically.
        return this.firstName() + " " + this.lastName();
        }, this);
    };
    ko.applyBindings(new ViewModel("Arne", "Franken"));
</script>

Run

  • ko.observable: KO fires events if variable values change
  • ko.computed: KO will update value if observables change
  • ko.computed(function(){},this): bind(this) correctly...

Simple List

<form data-bind="submit: addItem">New item:
    <input data-bind='value: itemToAdd, valueUpdate: "afterkeydown"' />
    <button type="submit" data-bind="enable: itemToAdd().length > 0">Add</button>
    <p>Your items:</p>
    <select multiple="multiple" width="50" data-bind="options: items"></select>
</form>
<script type="text/javascript">
    var SimpleListModel = function(items) {
        this.items = ko.observableArray(items);
        this.itemToAdd = ko.observable("");
        this.addItem = function() {
            if (this.itemToAdd() != "") {
                this.items.push(this.itemToAdd());
                this.itemToAdd("");
            }
        }.bind(this);
    };
    ko.applyBindings(new SimpleListModel(["Alpha", "Beta", "Gamma"]));
</script>

Run

  • this.items.push(this.itemToAdd());: Adds the item. Writing to the "items" observableArray causes any associated UI to update.
  • this.itemToAdd(""): Clears the text box, because it's bound to the "itemToAdd" observable
  • function(){}.bind(this): make sure to bind(this) correctly...

Interactive List - JS

var BetterListModel = function () {
    this.itemToAdd = ko.observable("");
    this.allItems = ko.observableArray(["Fries", "Eggs Benedict", "Ham", "Cheese"]);
    this.selectedItems = ko.observableArray(["Ham"]);
    this.addItem = function () {
        if ((this.itemToAdd() != "") && (this.allItems.indexOf(this.itemToAdd()) < 0))
        this.allItems.push(this.itemToAdd());
        this.itemToAdd(""); // Clear the text box
    };
    this.removeSelected = function () {
        this.allItems.removeAll(this.selectedItems());
        this.selectedItems([]); // Clear selection
    };
    this.sortItems = function() {
        this.allItems.sort();
    };
};
ko.applyBindings(new BetterListModel());

Run

  • add sort and remove functionality to ViewModel
